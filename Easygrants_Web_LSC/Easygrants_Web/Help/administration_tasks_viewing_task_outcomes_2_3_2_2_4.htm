Selecting the Grant Tasks

   

To select the tasks to be available to the public and staff users from the Grant File:

 

1.Click on the Add button in the Grant Task Information section of the Grant Tasks Workflow Builder page. The Workflow Builder - Task Editor page is displayed.
2.Select the Task (required). The task drop-down consists of all tasks with a type of grant task or internal review.
3.Enter the Description (required). The description will be displayed on the Home page.
4.Modify the Default Status as needed. The default status will be Incomplete.
5.Select the Default Outcome (required).
6.Select the Submitted Outcome (required).
7.Select the Assign To (required). The Assign To drop-down consists of the public user, contact roles (e.g., Mentor, Contracts Official), the primary staff affiliated with the grantee project, or the existing workgroups in Easygrants.
8.Enter the task dates, Start, Open, Due, Close, and End (required).

 

•Start Date: The date that the task will be displayed on the Home page. The task will not be enabled.
•Open Date: The date that the task is available to access.
•Due Date: The date that is published as the deadline for the task submission.
•Close Date: The date that the task is no longer available to access. It still displays on the Home page.
•End Date: The date the task no longer displays on the Home page.

 

9.Check if this is the initial task in the workflow. The initial task is the starting point for the grant lifecycle. There can only be one initial task for the workflow.
10.Click on the Save button to save the information and remain on the Workflow Builder - Task Editor page.
11.Click on the Save and Close button to save the information and return to the Grant Tasks Workflow Builder page.
12.To return to the Grant Tasks Workflow Builder page without saving the changes, click on the Close button.

 

Grant tasks can have associated compiled PDFs (PDF versions of the full submission).

 

To associate the PDF with the selected task:

 

1.From the Workflow Builder - Task Editor page, click on the Add button in the PDF Information section. The PDF Report Editor page is displayed.
2.Enter the PDF Description (required).
3.Select the PDF Key (required).
4.Select the PDF Type (required).
5.Check if generated PDFs should be locked. PDFs are typically locked after the closure of the phase to retain the submission information.
6.Click on the Save button to save the information and remain on the PDF Report Editor page.
7.Click on the Save and Close button to save the information and return to the Workflow Builder - Task Editor page.
8.To return to the Workflow Builder - Task Editor page without saving the changes, click on the Close button.

 

Saving the page without entering the required information or with incorrect formatting will display page level validation in red text at the top of the page. Required fields are marked with a red asterisk.

 

To edit the tasks to be available to the public and staff users from the Grant File, click on the Edit button next to the desired record in the Grant Tasks Information section of the Grant Tasks Workflow Builder page. The Workflow Builder -  Task Editor page is displayed with the selected record information. Use the steps above to complete the modification of the task.

 

To delete a task to be included in the grant tasks list:

 

1.Click on the Delete button next to the desired record in the Grant Task Information section of the Grant Tasks Workflow Builder page.
2.Click on the OK button in the confirmation pop-up to continue with the deletion. The page refreshes and no longer displays the deleted record.

 

The Delete button is only available for records that do not have active task assignments and are not being used in the workflow.
