Setting Review Cycle Conflicts and Preferences

 

   

The second step in managing the review cycle is to set the known review cycle conflicts and preferences. Setting the conflicts and preferences at the review cycle level is optional and can be done in bulk using a set process or manually by the participant or reviewer.

 

The conflicts and preferences set at the review cycle can be included in review stages. Refer to the Managing Review Stage Assignments section for more information.

 

Reviewers can also self-report conflicts and preferences as a review stage within the review cycle.

 

 

To access the Conflicts and Preferences section, click on the Reviews link in the left navigation. The Reviews Easylist page is displayed. Click on the Review Processing link in the left navigation. The Review Cycles page is displayed. Click on the Manage button for a specific review cycle. The Participant/Committee Pool page is displayed. Click on the Conflicts and Preferences tab. The Conflicts and Preferences Processing page is displayed.
