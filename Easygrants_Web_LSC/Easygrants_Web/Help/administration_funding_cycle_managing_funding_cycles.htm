Adding a Funding Cycle

 

   

To add a funding cycle:

 

1.Click on the Add button from the Funding Cycle Management section of Program Management page. The Funding Cycle Editor page is displayed.
2.Enter the Funding Cycle Name (required).
3.Enter the Technical Support E-mail Address (required).
4.Enter the Program Support E-mail Address (required).

 

The Program Support E-mail Address will be the address available in the system footer when in a particular funding cycle task.

 

5.Select the Time Period/Award Cycle.
6.Select the Fund Name.
7.Enter the Award Term (Years).

 

The Award Term (Years) determines how many years are displayed in the Budget grid for the budget entry. This is from a minimum of one year to a maximum of five years.

 

8.Enter the Start Date (required).
9.Enter the End Date (required).

 

The start and end dates for a Funding Cycle determine when the cycle will be available for new users to apply for funding for the cycle. If the cycle is not publicly available, the start and end dates should be set to past dates.

 

10. Enter the Total Budget Limit (required). This will limit the total budget entry by an applicant during the application/proposal phase when using the Easygrants budget grid.
11. Enter the Indirect Cost Limit (required). This will limit the percentage of indirect costs entry compared to the direct costs by an applicant during the application/proposal phase when using the Easygrants budget grid.
12. Select the Automatic Payment Generation information. Selecting Yes for Generate Payments Automatically will create the payment schedule when participants are awarded from the Outcomes page for a Review Stage. If selecting Yes, select if the default payment generation schedule or a custom generation process should be used. The default payment generation schedule is set up after a Funding Cycle is saved.
13. Enter the Automatic Grantee Report Generation information. Selecting Yes for Generate Grantee Reports Automatically will create the grantee reporting schedule when participants are awarded from the Outcomes page for a Review Stage. If selecting Yes, select if the default grantee report generation schedule or a custom generation process should be used. The default grantee report generation schedule is set up after a Funding Cycle is saved.
14.Click on the Save button to save the information and remain on the Funding Cycle Editor page.
15.Click on the Save and Close button to save the information and return to the Program Management page.
16.To return to the Program Management page without saving the changes, click on the Close button.

 

Saving the page without entering the required information or with incorrect formatting will display page level validation in red text at the top of the page. Required fields are marked with a red asterisk.

 

To view the funding cycle workflow:

 

1.Click on the View button next to the desired Funding Cycle in the Funding Cycle Management section of the page.
2.The page refreshes and displays the workflow builder tabs for each grant lifecycle phase starting with the Eligibility Workflow Builder.

 

To return to the Funding Cycles page, click on the Funding Cycle Management link in the History trail.

 

To edit a funding cycle, click on the Edit button next to the desired Funding Cycle in the Funding Cycle Management section. The Funding Cycle Details page is displayed with the selected record information. Click on the Edit button in the first section of the Funding Cycle Details page. The Funding Cycle Editor page is displayed. Use the steps above to complete the modification of the funding cycle.

 

To delete a funding cycle:

 

1.Click on the Delete button to the right of the desired Funding Cycle in the Funding Cycle Management section of the System Management page.
2.Click on the OK button in the confirmation pop-up to continue with the deletion. The page refreshes and no longer displays the deleted record.
