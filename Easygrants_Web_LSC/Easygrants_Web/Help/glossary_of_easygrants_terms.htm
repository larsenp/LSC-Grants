Glossary of Easygrants Terms

 

   

A

Action: List of features that can be used to interact with a particular individual or task.

 

C

Categories: Contact categorization features are available to assist users in organizing contact records stored in the system.

 

Custom Reports: Reports created by Altum that are set in the system. Report criteria are selected by the user at run time.

 

E

Easygrants Manager: The Staff and Admin module in Easygrants.

 

Easylist: Quick Search pages specific to each main section in the Easygrants Manager (e.g., Application Easylist, Grants Easylist).

 

Event: The completion of a task (e.g., Submit Proposal) or date criteria (e.g., deadline date) that causes a specific action in the system (e.g., automatic e-mail message sent to the applicant upon submission.

 

F

Funding Cycle: An instance of a funding opportunity in a certain time frame (e.g., Research 2008, Councils 2008).

 

Funding Opportunity: Second level in the Easygrants hierarchy allowing the organization of funding into program areas or award mechanisms (i.e., Research, Councils).

 

H

Home Page: The landing page for Easygrants users.  The Home page for Public users is different from the Staff and Administrative users.

 

L

Login Type: Level of access to the Easygrants system (i.e., Staff Member, Administrative, Public).

 

O

Outcome: The results of a particular task (i.e., Submitted, Not Submitted, Pending).

 

P

Program: Top level in the Easygrants hierarchy allowing the organization of funding into strategic giving areas.

 

Q

Queries: Configuration by the user to pull system data for Excel Export, HTML View, or to communicate by e-mail to a designated contact.

 

S

Status: The state of a given WfTA (i.e., Complete, Incomplete, Not Active).

 

T

Task: To Do item for a particular grant (i.e., Application, Review, Progress Report).

 

U

User: Individual accessing the Easygrants system (i.e., Staff, Applicant, Reviewer, Grantee, etc.) There are two over-arching categories of users – Public and Internal.

 

W

WfTA: Workflow Task Assignment

 

Workflow Task Assignment: WfTA – An instance of a Task being assigned to a person.

 

Workgroups: Grouping of staff and administrative contacts responsible for the same function. Task Assignments can be made to a Workgroup.
