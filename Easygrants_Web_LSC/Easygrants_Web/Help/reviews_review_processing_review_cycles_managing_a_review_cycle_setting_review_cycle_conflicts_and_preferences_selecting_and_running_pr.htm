Selecting and Running Preprocessing

 

   

Review Cycle conflicts and preferences can be assigned in bulk using the preprocessing options.

 

To assign conflicts and preferences in bulk from the Conflicts and Preferences Processing page:

 

1.Select the Process from the drop-down. The page will refresh and display the description of the selected process.

 

The preprocessing options include:

               

•Auto Generated Conflicts and Preferences: The process compares the reviewer's organization with that of the participant. If there is a match, a conflict record is created. All of the conflicts created within this process will have a source of "Internal - Auto".
•Clear Conflicts and Preferences Process Results: This process will clear all conflicts/preferences and matching records created in this cycle with a source of "Internal-Auto". Once this process has successfully executed you may select another conflicts and preferences process to run from the drop-down menu. This process is generally used to clear results if they have run a process by mistake.
•Copy Conflicts and Preferences from Previous Cycle: This process copies over any conflicts or preferences entered during previous review cycles. These conflicts and preferences may have been entered by reviewers, staff, or auto generated.

 

2.Click on the Go button to execute the process.

 

The Preprocess Generation Status section displays the processes run for the review cycle with the most recent request date, the user that made the request, and the current status of the process. To view the latest status, use the refresh button on your browser. The process will continue to run even if you move away from the page or close your browser.
