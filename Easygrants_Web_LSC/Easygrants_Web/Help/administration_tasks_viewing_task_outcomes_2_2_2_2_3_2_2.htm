Setting up the Grantee Reporting Workflow

 

   

Once the tasks have been selected, the next steps and communication can be set up based on the results, or outcome, of the submission. The workflow is set up by selecting the current task and the outcome for which the next task or e-mail should be sent. This section does not display until a task has been associated with the phase.

 

Example: The grantee (user) submits the annual progress report. The submission of this task sends an e-mail to the user, to designated staff on a workgroup responsible for reviewing all letters submitted, and creates the internal review task for the same workgroup.

 

Each branch of the workflow will need to be added to the Grantee Reporting Workflow section. For the grantee reporting phase, the workflow should include both the grantee reporting type tasks and the related internal review tasks.

 

To create new workflow for the grantee reporting phase:

 

1.Click on the Add button in the Grantee Reporting Workflow section. The Workflow Builder - Workflow Editor page is displayed.
2.Select the Task (required). The task drop-down will contain the tasks added previously. If a task has not been added, click on the Close button to return to the Grantee Reporting Workflow Builder page to add the task. The page will refresh after the selection of the task.
3.Select the Outcome (required). The outcome is set when the user submits the task. Note: Grantee Reporting type task outcomes consist of the submission task outcomes as well as outcomes based on review, e.g., Submitted, Not Submitted, Approved, etc.
4.Enter the Advanced Filter as needed.

 

The advanced filter field is used to insert XML (code) into the workflow to handle alternative workflow based on data or other criteria outside of the outcomes. For example, for submitted proposals where the budget is less than $1,000,000.00, no internal review task is required. For proposals with budgets greater than $1,000,000.00, an internal review task is created for a particular workgroup.

 

5.Select the Next Task (required). The task drop-down will contain all tasks associated with the grantee reporting phase. If no task is to be created but an e-mail is to be sent for the selected outcome, select N/A. The page will refresh and display the selected task information including the assign to type.
6.Click on the Add button in the Submission Communication section to add e-mail communications as part of the workflow. The Workflow Builder - E-mail Editor page is displayed.
7.Select the E-mail Template (required). Once selected, click on the View button next to the template. The e-mail template with the content is displayed in a new window.
8.Select the Recipient. The Recipient drop-down consists of the public user, the primary staff affiliated with the grantee project, or the existing workgroups in Easygrants.

 

The "To" field (merge code) in the selected e-mail template must match the selected recipient in order for the e-mail to be sent properly. Refer to the E-mail Templates section for more details on creating e-mails using merge codes.

 

9.Click on the Save button to save the information and remain on the Workflow Builder - E-mail Editor page.
10.Click on the Save and Close button to save the information and return to the Workflow Builder - Workflow Editor page.
11.To return to the Workflow Builder - Workflow Editor page without saving the changes, click on the Close button.

 

Click the Add button again to continue adding e-mail communications for the selected outcome.

 

12.Click on the Save button to save the workflow information and remain on the Workflow Builder - Workflow Editor page.
13.Click on the Save and Close button to save the workflow information and return to the Grantee Reporting Workflow Builder page.
14.To return to the Grantee Reporting Workflow Builder page without saving the changes, click on the Close button.

 

Saving the page without entering the required information or with incorrect formatting will display page level validation in red text at the top of the page. Required fields are marked with a red asterisk.

 

To edit existing workflow for the grantee reporting phase, click on the Edit button next to the desired record in the Grantee Reporting Workflow section of the  Grantee Reporting Workflow Builder page. The Workflow Builder -  Workflow Editor page is displayed with the selected record information. Use the steps above to complete the modification of the task.

 

To disconnect the workflow:

 

1.Click on the Disconnect button next to the desired record in the Grantee Reporting Workflow section of the Grantee Reporting Workflow Builder page.
2.Click on the OK button in the confirmation pop-up to continue with the removal of the workflow record. The page refreshes and no longer displays the record.

 

Disconnecting the outcome workflow will remove the selected "branch" from the workflow. Any submissions corresponding to the disconnected outcome will still remain in the system.
