Managing Conflicts and Preferences by Reviewer

 

   

Conflicts and preferences can be viewed, set, and updated by reviewer within the View or Modify Conflicts and Preferences section of the Conflicts and Preferences Processing page of the review cycle.

 

To view a conflict or preference for a reviewer:

 

1.Select the Reviewer name next to the View and Modify By Reviewer field.
2.Click on the Go button. The Conflicts and Preferences by Reviewer page is displayed.

 

The Conflicts and Preferences page displays the reviewer information including the name, the primary organization, and the related committee, and the current conflicts and preferences in the Conflicts and Preferences Details by Reviewer section with the following columns:

 

•Participant
•Participant Organization
•Conflicts/Preferences
•Type
•Source

 

The source of the conflict or preference is listed as one of the following:

 

Internal Auto: The conflict/preference records that were generated based on the Conflicts and Preferences automatic process (on the Conflicts and Preferences Processing page).

 

Internal Manual: The conflict/preference records that were created and edited by staff members.

 

External: The conflict/preference records that were created and edited by Reviewers.

 

The results can be sorted by the column headers. Clicking once will sort the list in ascending alpha or numeric order. Clicking on the column header a second time will sort the list in descending alpha or numeric order.

 

In the Conflicts and Preferences by Reviewer section, new conflicts and preferences can be added or existing conflicts and requests can be modified.

 

To return to the Conflicts and Preferences Processing page for the review cycle, click on the Close button.

 

To add a conflict or preference for a reviewer:

 

1.Click on the Add button on the Conflicts and Preferences Details by Reviewer section. The Conflicts and Preferences by Reviewer Editor page is displayed.
2.Select the Participant Name from the drop-down (required). The page refreshes and displays the Type and the Conflict/Preference Type.

 

If selecting a participant where a conflict or preference has already been identified, the page will display the current conflict and preference information.

 

3.Select the Type (required). The type will default to None. Select Preferred, Able, Not Able, or Conflict.
4.Select the Conflict Type (e.g., Financial, Organizational, or Personal).
5.Click on the Save button to save the changes and remain on the Conflicts and Preferences by Reviewer Editor page.
6.Click on the Save and Close button to save the changes and return to the Conflicts and Preferences by Reviewer page.
7.To return to the Conflicts and Preferences by Reviewer page without saving changes, click on the Close button.

 

Saving the page without entering required information or with incorrect formatting will display page level validation in red text at the top of the page. Required fields are marked with a red asterisk.

 

To edit a conflict or preference for a reviewer, click on the Edit button next to the desired record in the conflict or preference. The Conflicts and Preferences  by Reviewer Editor page is displayed with the selected record information. Use the steps above to complete the modification of the conflict or preference.

 

To remove a conflict or preference:

 

1.Click on the Edit button to the right of the desired record in the conflict or preference.The Conflicts and Preferences  by Reviewer Editor page is displayed.
2.Modify the Type to Able and the Conflict/Preference Type to <None> to remove the conflict or preference.
3.Click Save to save your changes and remain on the Conflicts and Preferences  by Reviewer Editor page.
4.Click Save and Close to save your changes and return to the Conflicts and Preferences by Reviewer page.
5.Click Close to return to the Conflicts and Preferences by Reviewer page without saving your changes.
