New Import

 

   

To import the new file into Easygrants:

 

1.Click on the Import New File button in the New Import section of the Import Management page. The New Import page is displayed.
2.Select the Import File Type. See below for descriptions of the file types provided in Easygrants.
3.Click on the Browse button to open the file Choose File Selector to select the files from the network or user's computer. The file should be an Excel file type.

 

All Import files must have an extension of .xls or .xlsx (MS Excel file extension).

 

4.Click on the Upload button to upload the selected file.

 

Easygrants provides a View option before you commit to uploading your file on the Current Import Requests page.

       

Easygrants Import File Format Requirements

 

The Import Process in Easygrants requires a specific file format for each type of file that is being imported into Easygrants.  Each file type format may vary slightly. All import files must have an .xls or .xlsx extension (MS Excel file type extension).

 

Easygrants provides the following import file types:

 

•Payments

 

Payments File Type

 

The Payments file type allows users to import payment data from accounting software, e.g., QuickBooks or Great Plains, to the payment records in Easygrants.

 

The data file may originate from your accounting software package and should adhere to the following:

 

1.MS Excel File Type Extension (.xls or .xlsx).
2.No formatting in the file (including Freeze Panes, Totals, Blank Columns or Rows, etc.).
3.Data should only be on Sheet 1 of the Excel Workbook.
4.The following column headers in the following order: Easygrants Payment ID; Related Easygrants Payment ID; Easygrants Grantee Project ID; Payment Date; Check Number; Payment Amount.
5.The Easygrants Grantee Project ID, Payment Date, and Payment Amount fields cannot contain a null value (i.e., the field must contain data).
6.At least one of the following is required: Easygrants Payment ID or Related Easygrants Payment ID.
•If there is a value in the “Easygrants Payment ID” column then the values in the “Payment Date,” “Check Number,” and “Payment Amount” columns are used to update the Payment table for the Easygrants Grantee Project ID/Easygrants Payment ID combination. If the row does not exist, a new record is not created.
•If there is a value in the “Related Easygrants Payment ID” column and the “Easygrants Payment ID” column is empty, then the record is processed as a refund/rescind/void and a new Payment record is created.
