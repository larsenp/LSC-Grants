Narrowing Query Results with Filters

 

   

The Filters section of the Query Tool Editor page allows you to narrow the report results set by creating conditions. When the system executes the query, it evaluates each field for which a filter value has been entered. All filter values must be true to be included in the report output. A filter consists of a Boolean operator (i.e., And, Or), a report field, a comparison operator (i.e., =, <>, >, <, Contains), and a value. The Boolean operator allows you to logically group filters.

 

To add filters to the query:

 

1.Enter a Boolean value (And, Or).

 

A Boolean value is not necessary if you are using only one filter.

 

2.Select the field to query on.
3.Select the operator (=, >, <, <>, or contains).

 

The ≤ and ≥ operators are not currently available as filters. If you would like to filter in this manner, select < or > and the desired value plus or minus one. For example, if you want to search for all grants less than or equal to $25,000, select “<” and “$25,001” as your operator and value.

 

4.Enter a value to compare to the chosen field in the Value text box.

 

If you are unsure of the value for a particular field, add that field to the selected list box and run the query to HTML without the query. The values will be displayed in the results.

 

5.Click on the Add button to add the filter to the query. The page refreshes and displays the new filter specifications in the Filter section.
6.Click on the Edit button next to a selected filter to modify the filter.
7.To delete a filter, click on the Delete button next to a selected filter.
