Selecting and Running Bulk Assignments

 

   

Easygrants supports the creation of bulk (i.e., automatic) assignments of submissions to reviewers based on the number of reviewers indicated for each task submission and on the conflicts and preferences determined during the review cycle setup.

 

To generate bulk review assignments:

 

1.Select the Assignment Process from the drop-down in the Bulk Assignments - Participants to Reviewers section of the Review Stage Assignments page. The page refreshes and displays a description of the selected assignment process.

 

The preprocessing options include:

               

•Generic Bulk Assignment: This assignment process will assign reviewers to each submission based on the number of reviewers desired per submission. If you wish to avoid making assignments where there is a known conflict of interest, the preprocess must be run and the request and conflicts box must be checked. The purpose of making these assignments is so that reviewers can use the system to self-report any conflicts and preferences or enter scores and comments. This process will typically be run during the Request and Conflicts with the request and conflict box left unchecked, so that all applicants may be assigned to all reviewers.

 

2.Click on the Include Conflicts and Preferences Pre-Processing? checkbox if you would like to include the conflicts and preferences determined during the review cycle setup.
3.Enter the desired number of review assignments per submission. To assign all reviewers to all submissions, enter “-1” in this field.
4.Click on the Go button for the Run Selected Assignment Process to generate the bulk assignments. The system will generate the assignments of reviewers to the submissions.

 

The Bulk Assignment Process Status section displays the processes run for the review stage with the most recent request date, the user that made the request, and the current status of the process. To view the latest status, use the refresh button on your browser. The process will continue to run even if you move away from the page or close your browser.
