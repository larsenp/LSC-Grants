Setting up the Related Third Party Workflow

 

   

As part of the grantee reporting phase, the related third party workflow must also be set up. Third party workflow is created when the grantee reporting task submission includes other contact submissions.

 

Example: As part of the grantee reporting submission, the grantee is required to provide two letters of ongoing support. One of the forms used in the progress report requires the grantee to verify existing or enter new references for the support letters. Once added, the grantee then notifies the person. The person receives an e-mail with login credentials for Easygrants. The reference logs into Easygrants and submits their ongoing letter of support online. An e-mail notification is then sent to the grantee to notify them that the requirement has been met.

 

Each branch of the workflow will need to be added to the Related Third Party Workflow section.

 

To create new workflow related to the third party tasks in the grantee reporting phase:

 

1.Click on the Add button in the Related Third Party Task Workflow section. The Workflow Builder - Related Task Workflow page is displayed.
2.Select the Parent Task (required). The task drop-down will contain the grantee reporting type tasks added previously. If a task has not been added, click on the Close button to return to the Grantee Reporting Workflow Builder page to add the task.
3.Select the Task (required) in the Related Task Information section. The task drop-down will contain the third party type tasks added previously. If a task has not been added, click on the Close button to return to the Grantee Reporting Workflow Builder page to add the task. The page will refresh after the selection of the task.
4.Click on the Add button in the Notification Communication section to add e-mail communications sent as notification of the task requirement for the third party role. The Workflow Builder - Related Third Party Tasks - E-mail Editor page is displayed.

 

The Notification Communication section of the Workflow Builder - Related Task Workflow allows the set up of the e-mails that are sent to the third party contact's e-mail when the grantee clicks the Notify button on the third party page in the grantee report.

 

5.Select the E-mail Template (required). Once selected, click on the View button next to the template. The e-mail template with the content is displayed in a new window.
6.Select the Recipient (required). The Recipient drop-down consists of the applicant and third party roles (e.g., Referee, Mentor, Sponsor) set up in Easygrants.

 

The "To" field (merge code) in the selected e-mail template must match the selected recipient in order for the e-mail to be sent properly. Refer to the E-mail Templates section for more details on creating e-mails using merge codes.

 

7.Select the Action (required).
8.Click on the Save button to save the information and remain on the Workflow Builder - Related Third Party Tasks - E-mail Editor page.
9.Click on the Save and Close button to save the information and return to the Workflow Builder - Related Task Workflow page.
10.To return to the Workflow Builder - Related Task Workflow page without saving the changes, click on the Close button.
11.Click on the Add button in the Submission Communication section to add e-mail communications sent as notification of the related third party task submission. The Workflow Builder - Related Third Party Tasks - E-mail Editor page will be displayed.
12.Select the E-mail Template (required). Once selected, click on the View button next to the template. The e-mail template with the content is displayed in a new window.
13.Select the Recipient (required). The Recipient drop-down consists of the applicant, the primary staff affiliated with the grantee project, or the existing workgroups in Easygrants.

 

The "To" field (merge code) in the selected e-mail template must match the selected recipient in order for the e-mail to be sent properly. Refer to the E-mail Templates section for more details on creating e-mails using merge codes.

 

14.Click on the Save button to save the information and remain on the Workflow Builder - Related Third Party Tasks - E-mail Editor page.
15.Click on the Save and Close button to save the information and return to the Workflow Builder - Related Task Workflow page.
16.To return to the Workflow Builder - Related Task Workflow page without saving the changes, click on the Close button.

 

Click the Add button again to continue adding e-mail communications for the selected outcome.

 

17.Click on the Save button to save the workflow information and remain on the Workflow Builder - Related Task Workflow page.
18.Click on the Save and Close button to save the workflow information and return to the Grantee Reporting Workflow Builder page.
19.To return to the Grantee Reporting Workflow Builder page without saving the changes, click on the Close button.

 

Saving the page without entering the required information or with incorrect formatting will display page level validation in red text at the top of the page. Required fields are marked with a red asterisk.

 

To edit existing workflow for related third party tasks in the grantee reporting phase, click on the Edit button next to the desired record in the Related Third Party Task Workflow section of the Grantee Reporting Workflow Builder page. The Workflow Builder -  Related Task Workflow Editor page is displayed with the selected record information. Use the steps above to complete the modification of the task.

 

To disconnect the workflow:

 

1.Click on the Disconnect button next to the desired record in the Related Third Party Task Workflow section of the Grantee Reporting Workflow Builder page.
2.Click on the OK button in the confirmation pop-up to continue with the removal of the workflow record. The page refreshes and no longer displays the record.

 

Disconnecting the outcome workflow will remove the selected "branch" from the workflow. Any submissions corresponding to the disconnected outcome will still remain in the system.
