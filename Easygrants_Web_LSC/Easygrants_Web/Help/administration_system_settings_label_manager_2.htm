Event Manager

 

   

The Event Manager feature allows the administrator to specify advanced data manipulation and updates ("events") that can be triggered via a set of workflow events (e.g. prior to submitting a task, after a task assignment is created, etc.). Events that have been created are available in the Event Manger to enable the administrator to designate descriptions and status for each event.

 

Examples of events include:

 

•Updating the Grant Status once a proposal/application has been awarded
•Automatically creating the grantee reporting schedule once a proposal/application has been awarded

 

Events are created by a technical resource on your implementation team and made available via the Event Manager.

 

Several types of events are available via the event manager. They include:

 

 

Event Type

Definition

Post-Assignment

This event type is used when the main purpose of the process is to update data. This type is available to be configured via the Task Information Editor within the Workflow Builder. This event is triggered when a task has been assigned to an individual or a group.

Post-Submission

This event type is used when the main purpose of the process is to update data. This type is available to be configured via the Task Information Editor within the Workflow Builder. This event is triggered when a task has been submitted.

Post-Submission Workflow

This event type is used when the main process that is needed is tied to business rules (combination of workflow task and outcome). It is configured via the Workflow Builder Editor of the Workflow Builder. This event is triggered after a task has been submitted and after any Post-Submission workflow events.

Pre-Assignment

This type is used when the main process that is needed will decide whether the task should be assigned. It can also be used to pre-load data or copy data from task to task. This type is available to be configured via the Task Information Editor within the Workflow Builder. This event is triggered before a task is assigned.

Pre-Submission

This type is used when the main process that is needed is needed for more complicated business rules validation. This type is available to be configured via the Task Information Editor within the Workflow Builder. This event is triggered before a task is submitted.
