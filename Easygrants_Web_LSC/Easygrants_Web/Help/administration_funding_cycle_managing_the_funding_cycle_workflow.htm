Adding and Managing the Funding Cycle Workflow

 

   

For each Funding Cycle, a specific workflow, or path for the grant lifecycle, can be created using the Workflow Builder. This includes selecting the tasks and assigning the associated task dates for a funding cycle, determining the workflow (e.g., what happens next) for a funding cycle, and selecting the e-mail communication that is sent upon submission of a task.

 

The tasks can be assigned to types of users including public users (applicants, reviewers, grantees), the primary staff affiliated to the grantee project, or to workgroups.

 

Before creating the Funding Cycle Workflow, make sure to create the Tasks, Workgroups, and E-mail Templates that will be used!

 

For more information on affiliating staff members as the primary contact for a grant refer to the Affiliated Staff Management section. For more information on setting up new workgroups refer to the Adding a Workgroup section.

 

For more information on creating Tasks refer to the Task Builder section. For more information on creating an E-mail Template refer to the E-mail Templates section.

 

The Workflow consists of setting up the following phases and functionality for the grant lifecycle:

 

•Eligibility - Tasks allowing the pre-filtering of applicants for a particular funding cycle.
•Application - Tasks allowing the submission of applications or proposals for funding consideration, the submission of reference letters, or the submission of internal staff review decisions.
•Review - Tasks allowing for the committee or peer review of submissions by applicants or grantees.
•Grantee Reporting - Tasks allowing for the reporting on active or closed grants with either project progress or expenditures.
•Grant Tasks - Tasks allowing the primary contact/PI or staff to request amendments and changes to the grant.
•Validation - Rules for the submission of the tasks within Easygrants.

 

To access the Workflow Builder, click on the View button next to the desired funding cycle. The Workflow Builder page is displayed.

 

To view the full workflow at any time, click on the View Workflow button. The current tasks are displayed in a new window.
